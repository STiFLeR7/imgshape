# Design Spec: imgshape v4.2 "Performance & Polish"

## 1. Overview
imgshape v4.2 is an incremental update to the Atlas engine (v4.0+) that matures the GPU acceleration path and introduces semantic dataset drift detection. It bridges the gap between statistical analysis (v4.1) and autonomous agentic curation (planned for v5.0).

## 2. Goals
- **Stabilize GPU Acceleration:** Move from "best-effort" to "production-grade" PyTorch/CUDA kernels with better error handling and batching.
- **Drift 2.0 (Semantic):** Implement embedding-based drift detection using CLIP or DINOv2 to detect semantic shifts that histograms miss.
- **Plugin Expansion:** Add specialized fingerprinting profiles for `medical`, `satellite`, and `ocr` to support niche domains.
- **Cloud Native Support:** Artifact serialization optimized for Vertex AI and AWS SageMaker model registries.

## 3. Architecture Changes

### 3.1. GPU Kernel Optimization (`imgshape.fingerprint_v4`)
- **Current Issue:** v4.1 performs many small GPU transfers per image, leading to overhead.
- **v4.2 Solution:** Implement **Batched GPU Processing**. The `Atlas` orchestrator will collect images in batches (e.g., size 32) before transferring to VRAM.
- **Components:**
  - `BatchedGPUHandler`: Manages GPU memory pools and batch transfers.
  - `AcceleratedEntropy`: Optimized Shannon entropy kernel using `torch.histc`.

### 3.2. Semantic Drift Detection (`imgshape.compare_v4`)
- **Current Issue:** v4.1 uses histogram intersection (color/spatial), which fails if "cat" images are replaced by "dog" images of the same color distribution.
- **v4.2 Solution:** **Embedding-based Drift**.
- **Implementation:**
  - Integrate `open_clip` or `torchvision.models` (SqueezeNet/MobileNet for speed).
  - Compute a "Semantic Signature" (centroid of embeddings) during fingerprinting.
  - Use **Cosine Similarity** and **MMD (Maximum Mean Discrepancy)** between baseline and current embeddings to calculate `semantic_drift`.

### 3.3. Profile-Driven Fingerprinting (`imgshape.profiles`)
- Extend the `Profile` abstraction to allow domain-specific metrics:
  - `MedicalProfile`: DICOM metadata, HU units, slice consistency.
  - `SatelliteProfile`: GSD (Ground Sample Distance), cloud cover estimation.
  - `OCRProfile`: Text density, orientation variance, contrast ratios.

## 4. Data Flow
1. **Fingerprint Stage:**
   - Raw Images → `BatchedGPUHandler` → GPU
   - GPU → (Entropy, Blur, *NEW: Embeddings*)
   - Result → `DatasetFingerprint` (v4.2 schema)
2. **Comparison Stage:**
   - Baseline FP + Current FP → `DriftEngine`
   - `DriftEngine` → (Statistical Drift + *NEW: Semantic Drift*)
   - Result → `DriftScore` (v4.2)

## 5. Success Criteria
- **Performance:** 2x faster fingerprinting for datasets >10k images vs v4.1.
- **Accuracy:** Detect semantic shift (e.g., class swap) with >90% confidence in drift reports.
- **Compatibility:** Full backward compatibility with v4.0 and v4.1 fingerprints.

## 6. Testing Strategy
- **Benchmark Suite:** Compare CPU vs GPU batching speeds on COCO/ImageNet samples.
- **Drift Validation:** Synthetic drift test (Swap classes in a dataset, verify `semantic_drift > 0.5`).
- **Profile Tests:** Unit tests for new Medical/Satellite metric extraction.
